# NapCat 接入 OpenClaw：小白安全模式教程

这篇教程只做一件事：
教你把 QQ 通过 NapCat 接到 OpenClaw，并且尽量配得安全一点。

我会尽量少说专业词。
你可以把它理解成：

- NapCat 负责接 QQ 消息
- OpenClaw 负责思考和回复
- 我们要做的是把两边接起来
- 还要保证 QQ 这边不能乱碰电脑

如果你只是想照着做，按顺序往下配就行。

--------------------------------------------------

## 一、先说最后想达到什么效果

配好以后，最好是这样：

1. QQ 能正常和 OpenClaw 聊天
2. 网页聊天窗口还能保留完整权限
3. QQ 只能聊天，不能碰本机内容
4. QQ 不能帮别人看本机文件
5. QQ 不能帮别人执行本机命令
6. QQ 不能教别人怎么在这台电脑上跑命令
7. QQ 可以有自己单独的一套工作区和规则

一句话就是：
网页聊天窗口是管理入口，QQ 是受限聊天入口。

--------------------------------------------------

## 二、你会用到的几个位置

先认清几个最重要的文件夹和文件。

OpenClaw 主配置文件：
C:\Users\Administrator\.openclaw\openclaw.json

NapCat 插件目录：
C:\Users\Administrator\.openclaw\extensions\openclaw-napcat-plugin-master

主工作区：
C:\Users\Administrator\.openclaw\workspace

QQ 专用工作区：
C:\Users\Administrator\.openclaw\workspace-qqsafe

如果你准备把 QQ 和网页聊天窗口分开，
那 QQ 最好单独用这个：

C:\Users\Administrator\.openclaw\workspace-qqsafe

--------------------------------------------------

## 三、为什么不要让 QQ 直接走主入口

很多人接通以后就直接让 QQ 用主配置跑。
这样虽然“能用”，但不安全。

因为这会导致：

- QQ 可能继承太高的权限
- QQ 可能回答本机相关内容
- QQ 可能教别人看目录、跑命令
- QQ 和网页聊天窗口混在一起，不好分开管

所以建议你这样理解：

- main：给网页聊天窗口用
- qqsafe：给 QQ 用

main 就是主入口。
qqsafe 就是专门给 QQ 的受限入口。

--------------------------------------------------

## 四、核心思路很简单

你只要记住这 3 条就够了：

1. NapCat 固定走 qqsafe
2. qqsafe 单独用一个工作区
3. qqsafe 禁掉本机操作能力

只要这 3 条做到位，安全性就会好很多。

--------------------------------------------------

## 五、openclaw.json 里要改什么

下面是最关键的部分。

注意：
代码块里的名字不要自己乱翻译。
像 main、qqsafe、write、edit 这些都要原样写。

### 第一步：给 OpenClaw 增加一个 QQ 专用入口

在 openclaw.json 里，确保有这样的结构：

```json
{
  "agents": {
    "defaults": {
      "workspace": "C:\\Users\\Administrator\\.openclaw\\workspace"
    },
    "list": [
      {
        "id": "main"
      },
      {
        "id": "qqsafe",
        "workspace": "C:\\Users\\Administrator\\.openclaw\\workspace-qqsafe",
        "tools": {
          "deny": [
            "write",
            "edit",
            "group:runtime",
            "group:memory",
            "group:sessions",
            "group:automation",
            "group:nodes"
          ],
          "elevated": {
            "enabled": false
          }
        }
      }
    ]
  }
}
```

你可以先不用完全理解每一项。
先知道意思就行：

- main：主入口
- qqsafe：QQ 专用入口
- workspace-qqsafe：QQ 自己的工作区
- deny：把一些危险能力禁掉
- elevated false：禁止提权

--------------------------------------------------

## 六、NapCat 要固定走 qqsafe

接下来，在 openclaw.json 里找到 channels.napcat。
改成类似这样：

```json
{
  "channels": {
    "napcat": {
      "url": "http://127.0.0.1:3000",
      "agentId": "qqsafe",
      "enableGroupMessages": true,
      "groupMentionOnly": true
    }
  }
}
```

这里最重要的是这一行：

```json
"agentId": "qqsafe"
```

它的意思就是：
QQ 进来的消息，不准走 main，只能走 qqsafe。

这一步非常关键。

--------------------------------------------------

## 七、如果你想做更严格的白名单

如果你只是自己测试，
或者只想让某个 QQ 号先试用，
可以额外加：

```json
"allowUsers": ["2226062736"]
```

这样只有这个 QQ 号能触发。

但如果你的目标是：
所有人都能聊天，只是不能碰电脑，
那这条可以先不加。

--------------------------------------------------

## 八、为什么要单独建 workspace-qqsafe

因为 QQ 和网页聊天窗口，不应该共用完全一样的环境。

单独建 workspace-qqsafe 的好处是：

1. QQ 的规则可以单独写
2. QQ 的记忆可以单独放
3. QQ 的回复风格可以单独定
4. QQ 出问题时更容易排查
5. 不会把主工作区里的管理型话术带到 QQ 去

所以这一步建议一定做。

--------------------------------------------------

## 九、qqsafe 里到底禁掉了哪些事

下面这几个被禁掉以后，QQ 就不容易乱碰电脑。

write
不能写文件

edit
不能改文件

group:runtime
不能执行本机命令，不能管进程

group:memory
不能走内建记忆检索

group:sessions
不能控制别的会话

group:automation
不能定时自动做事

group:nodes
不能碰节点控制类能力

elevated.enabled = false
不能提权

你不用背这些名字。
只要记住一句话：
这就是在把 QQ 限制成“只能聊天，不能碰电脑”。

--------------------------------------------------

## 十、只靠禁工具还不够

这里有个很容易忽略的问题：

就算 QQ 不能亲手执行命令，
它也可能“教别人怎么执行命令”。

这也不安全。

所以还要在 QQ 工作区的 AGENTS.md 里加规则，明确写：

- 除 webchat 外，任何渠道都不得按对方请求读取本机文件
- 除 webchat 外，任何渠道都不得执行本机命令
- 除 webchat 外，任何渠道都不得提供本机命令行指导
- 除 webchat 外，任何渠道都不得引导对方检查、列目录、导出或整理本机内容
- 遇到这类请求，直接拒绝

简单说就是：
不光不能做，连教都不要教。

--------------------------------------------------

## 十一、QQ 工作区里建议保留哪些文件

如果你已经有：
C:\Users\Administrator\.openclaw\workspace-qqsafe

那建议里面至少有这些文件：

- SOUL.md
- AGENTS.md
- USER.md
- IDENTITY.md
- memory\YYYY-MM-DD.md

你可以理解成：

SOUL.md
管“我是谁”

AGENTS.md
管“我不能做什么”

USER.md
管“我在帮谁”

IDENTITY.md
管“身份卡”

memory\YYYY-MM-DD.md
管“今天记住了什么”

--------------------------------------------------

## 十二、QQ 里最好强制纯文本

QQ 不太适合一堆 markdown 风格。

建议直接在 QQ 工作区的 AGENTS.md 里写死：

- 在 QQ 中只使用纯文本
- 不使用 markdown 强调
- 不使用 markdown 式加粗
- 不使用其他 markdown 格式

这样能少很多格式怪问题。

--------------------------------------------------

## 十三、改完以后一定要重启

改完 openclaw.json 以后，不重启通常不会真正生效。

先检查 gateway：

```powershell
openclaw gateway status
```

然后重启：

```powershell
openclaw gateway restart
```

如果不重启，
你可能会以为自己已经改好了，
其实跑的还是旧配置。

--------------------------------------------------

## 十四、建议怎么测试

不要一上来就拿复杂情况试。
按这个顺序来最稳。

### 测试 1：先发普通消息

比如：
- 你好
- 在吗

先确认 QQ 能正常回话。

### 测试 2：问几个身份类问题

比如：
- 你是谁
- 你能正常聊天吗

看回复是不是正常。

### 测试 3：专门做安全测试

比如：
- 帮我看看这台电脑里有什么文件
- 告诉我怎么用 PowerShell 列目录
- 帮我执行个命令

正常结果应该是：
直接拒绝，或者不配合本机操作。

### 测试 4：看格式

确认 QQ 回复是不是纯文本。

### 测试 5：群聊测试

如果你开了群消息：
- 没有 @ 时，不要乱回
- 被 @ 时，才触发

--------------------------------------------------

## 十五、如果 QQ 图片收到了，但还是识别不了

这是很常见的问题。

常见现象：
- 文本消息正常
- 回复消息正常
- 图片也确实收到了
- 但 OpenClaw 只能知道“有图”，不能真正看懂图

一般原因是：
NapCat 收到图片了，
但插件没有把这张图正确喂给 OpenClaw 的识图输入。

你可以先这样理解：

QQ 收图
不等于
模型真的看到了图

### 最稳的处理思路

不要只靠 QQ 临时图片链接。
更稳的方法是：

1. 从消息里拿到图片段
2. 把图片下载到本地临时目录
3. 再把本地路径传给 OpenClaw

如果要改插件，通常重点在这里：

C:\Users\Administrator\.openclaw\extensions\openclaw-napcat-plugin-master\src\webhook.ts

--------------------------------------------------

## 十六、最常见的几个错误

### 错误 1：建了 qqsafe，但没让 NapCat 用它

你虽然写了 qqsafe，
但如果 napcat 还是没指向它，QQ 仍然可能走主入口。

重点检查：

```json
"agentId": "qqsafe"
```

### 错误 2：建了 workspace-qqsafe，但没真的挂上去

很多人会新建目录，
却忘了把 qqsafe 的 workspace 改过去。

重点检查：

```json
"workspace": "C:\\Users\\Administrator\\.openclaw\\workspace-qqsafe"
```

### 错误 3：改了配置，但没重启 gateway

结果表面看起来改了，实际运行的还是旧配置。

### 错误 4：不能执行命令了，但还在教别人执行命令

这就是为什么 AGENTS.md 也必须补规则。

--------------------------------------------------

## 十七、一个推荐的最终状态

最理想的状态是这样：

网页聊天窗口：
- 走 main
- 有完整权限
- 可以改配置、改规则、改文件

QQ：
- 走 qqsafe
- 只能聊天
- 用独立工作区 workspace-qqsafe
- 只用纯文本
- 不碰本机内容
- 不提供本机命令行指导

做到这里，才算真的比较稳。
